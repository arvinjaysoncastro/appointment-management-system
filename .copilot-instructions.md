## Copilot Instructions

Follow the architectural and engineering rules defined in:

* `docs/AI_ENGINEERING_RULES.md`
* `docs/AI_GUARDRAILS.md`

These documents define the **mandatory architecture constraints** for this repository.

### Core Principles

When generating or modifying code, always follow these rules:

* Follow **Clean Architecture** dependency direction.
* Keep **controllers thin** — they should only handle HTTP concerns.
* Place **business logic in Application services**.
* The **Domain layer must not reference frameworks or infrastructure**.
* **Repository interfaces** belong in the Application layer.
* **Repository implementations** belong in the Infrastructure layer.
* Use **DTOs for communication between API and Application layers**.
* All IO operations must use **async / await**.
* Avoid **magic strings and magic numbers**.

### Safety Guidelines

* Respect the existing architecture and folder structure.
* Do not introduce new frameworks or patterns unless explicitly requested.
* Do not refactor unrelated code.
* Prefer **simple, readable implementations** over clever abstractions.
* Make the **smallest change necessary** to satisfy the requirement.
